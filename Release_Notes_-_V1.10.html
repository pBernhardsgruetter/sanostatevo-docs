Release Notes - V1.10

1.10.0 - 07.05.2018

Features

  • [SEP-733] - Einsatzerfassung / Bei mehreren Patienten pro Einsatz mit ELS3 können unterschiedliche SNZ-Einsatznummern vergeben werden (pro Patient ein Auftrag mit SNZ-Auftragsnummer).

  • [SEP-771] - Einsatzerfassung / Verknüpfung ELS3-Einsatz bei fehlender SNZ-Einsatznummer wurde überarbeitet/verbessert

  • [SEP-582] - Einsatzerfassung / Auf dem Einsatz wird ein neues Feld "Notarzt Einsatzende (Verrechnung)" bereitgestellt, damit die Endzeit der verrechenbaren Arbeitszeit erfasst werden kann,

  • [SEP-729] - Einsazerfassung / Situatives Löschen bei Adresstyp-Wechsel wurden komplett überarbeitet implementiert (Wechsel von Institution auf Patientenadresse resp. sonstiges und umgekehrt)

  • [SEP-734] - Pendenzen / Die Anzeige der Pendenzen auf dem Dashboard und der Ribbon-Bar stimmen nun überein

  • [SEP-348] - Med. Daten / Können neu "Defi-Pads" erfasst werden

  • [SEP-350] - Med. Daten / Es können Reanimationsdaten erfasst werden (Reanimation erfolgt Ja/nein + die wichtigsten Zusatzdaten). Diese sind Pflichtfelder, wenn Diagnose = "Herz-Kreislauf-Stillstand"

  • [SEP-639] - Med. Daten / Intoxikation (Unfall) und Intoxikation (Krankheit) mit Detailfeld versehen, damit die Art der Intoxikation dokumentiert werden kann

  • [SEP-660] - Med. Daten / Verbesserung der Körperregionen zu Fraktur ist umgesetzt.

  • [SEP-663] - Med. Daten / Für Einsätze ohne Diagnose kann in einem Auswahlfeld "Keine Diagnose, Transportgrund" dokumentiert werden, warum keine Diagnose erfasst werden kann. Dieses Feld übersteuert allfällige Diagnose-Pflichtfelder

  • [SEP-665] - Med. Daten / Atmung, Sauerstoff erhält den neuen Eintrag "Inhalationsmaske".

  • [SEP-666] - Med. Daten / Atemweg-Management erhält neue Einträge und Textanpassungen.

  • [SEP-667] - Med. Daten / Medikamentenprüfung mit Anästhesiekompetenz oder Notarzt mit Begründung. Über die Stammdaten können "kritische Medikamente" erfasst werden und es kann - kundenspezifisch - eine Begründung bei deren Erfassung verlangt werden

  • [SEP-680] - Med. Daten / Auswahlfeld "Bergung" wurde um "Roll-In-Trage" erweitert.

  • [SEP-682] - Med. Daten / Atemweg-Management wurde um die beiden Einträge "Thorax-Drainage" und "Thorax-Punktion" ergänzt

  • [SEP-745] - Med. Daten / Neues Feld "Blutentnahme" als Ja/Nein-Feld erstellt

  • [SEP-746] - Med. Daten / Im Detailfeld zu Diagnose RQW kann neu der Eintrag "Kopf" ausgewählt werden.

  • [SEP-747] - Med. Daten / Diverse Rechtschreibefehler in Diagnosen sind korrigiert

  • [SEP-752] - Med. Daten / Neurologie, Feld "Befund" -> Dysarthrie aufgenommen und Dysphasie deaktiviert.

  • [SEP-406] - Faktura / SOH - Für jeden Einsatz wird sowohl ein Abrechnungslauf für die SOH-Tarifstruktur wie auch für die Spezialstruktur 94XX-Tarifstruktur angezeigt.

  • [SEP-580] - Faktura / LST - Spezialprozesse (ohne FID, Notarzteinsatz, Rückverlegung in anderes Spital)

  • [SEP-581] - Faktura / LST - Für einen Einsatz mit Notarzt werden für den beteiligten Notarzt weitere Leistungen anhand der Einsatzzeiten (Tarmed-Regel) erzeugt.

  • [SEP-636] - Faktura / BAD - Für Sekundäreinsätze mit Sekundärleistung wird die Grundtaxe und Kilometerentschädigung mit Unterscheidung nach Aufgebotsstufe, resp. NACA-Code ermittelt.

  • [SEP-651] - Faktura / BAD - Für Primäreinsätze mit Grundtaxe und Zeitzuschlägen wird mit Unterscheidung nach Aufgebotsstufe (resp. Dringlichkeit) nach Krankenkassentarif abgerechnet, ausser es handelt sich um einen MTK/UVG-Tarif.

  • [SEP-652] - Faktura / BAD - Für Primäreinsätze mit Grundtaxe und Zeitzuschlägen wird mit Unterscheidung nach Aufgebotsstufe (resp. Dringlichkeit) nach Krankenkassentarif abgerechnet, sofern es sich um einen MTK/UVG-Tarif handelt.

  • [SEP-724] - Faktura / BAD - Für Primäreinsätze mit NEF-Einsatz wird die Verrechnung NEF und Notarzt verbessert.

  • [SEP-737] - Faktura / BAD - Für Primäreinsätze mit mehreren Patienten wird gemäss KK-Tarifkatalog eine Reduktionszeile ermittelt, damit die Kosten (exkl. Zusatzpaket) auf die Anzahl Patienten aufgeteilt wird.

  • [SEP-739] - Faktura / BAD - Es werden unterschiedliche ProviderID in Abhängigkeit des Fakturaparameters mit Unterscheidung zwischen intern und extern erzeugt, damit mit ALIS die richtige ProviderID (Kostenstelle) übermittelt werden kann.

  • [SEP-767] - Faktura / BAD - Für Sekundärtransporte mit mehreren Patienten wird immer eine fixe Reduktion pro Patient von CHF 150.00 erzeugt, unabhängig von der Anzahl Patienten.

  • [SEP-738] - Faktura / Transportweg - Ermittlung des Transportwegs über Parametrisierungen ermöglicht, damit diese per ALIS-Export ausgegeben werden können

  • [SEP-610] - Administration / Die Rollenzuordnung für Superuser ist überarbeitet (Stichwortsuche, verbesserte Zuordnung Rolle zu Mitarbeiter).

  • [SEP-611] - Administration / Die Berechtigungsobjekte für BST-Admin ist überarbeiter, damit die Zuordnung von Rollen zu Funktionen verbessert wird und kundenspezifische Rollen verwendet werden können

  • [SEP-526] - Rolle / Es wurden neue Beispieldaten Benutzerkontos erfasst.

  • [SEP-330] - Berichte / Stammdaten - Im Berichtsmodul für Stammdaten wurde die Diagnoseliste mit Suchfeldern (Einteilung, Kategorie, Bezeichnung,Synonym) ergänzt, damit sinnvoll gefiltert werden kann.

  • [SEP-787] - Berichte / Im Bericht "Dispositionsnachweis Zusammenfassung" werden bei den Prozentangaben zwei Kommastellen angegeben.

  • [SEP-509] - Auswertung / Nicht erreichte Ausrückdauer wurde überarbeitet

  • [SEP-616] - Validierung / BAD - GUI- und Validierungsregeln für Baden / Schwerpunkt "med. Daten"

  • [SEP-719] - Validierung / SOH - Neue Validierungsregeln (Teil April 2018) sind implementiert. Eine SNZ Nummer darf innerhalb eines Jahres nur einmal verwendet werden.

  • [SEP-768] - Validierung / BAD - Anpassungen in den Medizinischen Daten / Neurologie betreffend Lichtreaktion wurden erstellt.

  • [SEP-769] - Validierung / BAD - Anpassungen in den Medizinischen Daten / Neurologie betreffend dem Zusammenhhang von Neurologie (Unauffällig "Nein") und Psyche wurden erstellt.

  • [SEP-770] - Validierung / BAD - Anpassungen in den Medizinischen Daten / Neurologie, bei Unauffällig "Nein" wird der Blutzucker zum Pflichtfeld.

  • [SEP-778] - Validierung / BAD - Die Fakturadestination (Fakturierungsparameter 3) wird zum Pflichtfeld, wenn der Patient erfasst und verrechenbar ist.

  • [SEP-736] - Berechtigungen / BAD - GUI-Regeln im Bereich Faktura / Patient (Rolle Rechnungsverwaltung) sind angepasst

  • [SEP-618] - Patient / Angabe der Versicherung kann als kombiniertes Feld entweder als Auswahlfeld (aus Stammdaten) oder als Freittextfeld erfasst werden

  • [SEP-619] - Patient / Zum Patient kann in einem Freitextfeld die "Krankenkassen-Kartennummer" erfasst werden

  • [SEP-751] - Patient / Wenn keine Patientendaten vorhanden sind, sind FID und PID keine Pflichtfelder.

  • [SEP-748] - Patientenerfassung / Die Patientdaten sind Pflichtfelder, wenn in als Parameterfeld auf der Leistungsart nicht NoPat steht. Ohne dem Merker "NoPat" kann kein Patient zugefügt werden. Diese Erweiterungen wirkt sich auf die gesamte Parametrisierung aus

  • [SEP-720] - Business-Regeln / Verbesserung bezüglich "Fehleinsatz/Einsatzabbruch" mit Steuerung über Leistungsart-Merker und umgekehrt

  • [SEP-721] - Business-Regeln / SOH - Die Leistungsart "Verlegung innnerhalb soH" steuert die Fakturafelder.

  • [SEP-575] - AmbuPad / Der AmbuPad-Server soll nicht abgeholte XML vom ZOLL-Server abholen und in der Datenbank ablegen.

  • [SEP-657] - AmbuPad / Der SanostatEvo-Server stellt einen REST-Service für die Bereitstellung von neuen XML-Files bereit
