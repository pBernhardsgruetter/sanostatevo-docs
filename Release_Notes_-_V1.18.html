    Release Notes - V1.18

    V1.18.0 - 03.03.2020

    Features / Stories

    • [SEP-1548] - FlexSelect / Suche für Bemerkungshistorie

    • [SEP-1988] - FlexSelect / Filter nach 'Unvollständigen' AmbuPad-Protokollen ist möglich.

    • [SEP-1668] - Einsatzerfassung / Ein Einsatz kann als Report inkl. Bemerkungen und Bemerkungshistorie ausgegeben werden, eine Seite pro Patient (kundenspezifisch)

    • [SEP-1772] - Einsatzerfassung / Bei 'Einsatzabbruch = NEIN' darf in der Leistungsart nicht Einsatzabbruch/Fehleinsatz' stehen, ist korrigiert.

    • [SEP-1887] - Einsatzerfassung / Wenn der Rettungsdienst nur einen Stützpunkt, wird dieser automatisch gewählt.

    • [SEP-1737] - Med. Daten / Unter Kreislauf stehen im Sanostat Evo die Auswahlfelder analog AmbuPad zur Verfügung.

    • [SEP-1932] - Med. Daten / Alle Med. Daten als Pflichtfelder validieren (kundenspezifisch)

    • [SEP-1798] - AmbuPad / Der Originalname (Vorname/Nachname) muss beim Verknüpfen eine FID in einem Feld anzeigen, damit sichtbar ist, ob der Patient irrtümlich geändert wurde.

    • [SEP-1893] - AmbuPad / XSD-Wertevalidierung wie Blutzucker ist in XSD v5.0 eliminiert worden

    • [SEP-1905] - AmbuPad / Neues Feld "EO resp. ZO unterdrücken" im Stammdatenexport "locations.csv" berücksichtigen. damit nicht gewünschte Institutionen im AmbuPad nicht angezeigt werden.

    • [SEP-1949] - AmbuPad / In den Stammdaten der Spitäler steht neu eine alternative E-Mail-Adresse zur Erfassung zur Verfügung. Diese können als email_reg oder fax_reg exportiert und für die Patientenvoranmeldung im Ambulance Pad verwendet werden.

    • [SEP-1950] - AmbuPad / Neu wird ein Freitextfeld als Diagnosefeld unter dem Patient bereitgesetllt, damit auch Freitextdiagnosen aus dem Ambulance Pad übernommen werden können.

    • [SEP-1989] - AmbuPad / Die Übernahme 'Begründung' von nicht vollständigem AmbuPad-Protokoll ist umgesetzt.

    • [SEP-1662] - Faktura / Aufgebotstufen A-F einrichten und Abrechnung testen (kundenspezifisch)

    • [SEP-1722] - Faktura / 4. Zeile bei Rechnungsadresse (kundenspezifisch)

    • [SEP-1844] - Faktura / KLA-Regeln bei Patient transportiert = JA, dann ist unter Bergung/Lagerung die Transportart 'Liegend' oder 'Sitzend' ein Pflichtfeld und wird validiert.

    • [SEP-1881] - Faktura / Im Einsatz Notarzt auf NEF und auf RTW, dann auf RTW den NA nicht verrechnen (kundenspezifisch)

    • [SEP-1961] - Faktura / Abrechnung trotz fehlender FID ermöglichen, LeiSchlüssel 'FakturaOhneFidErlauben' anwenden (kundenspezifisch)

    • [SEP-1996] - Faktura / Notarzt-Verrechnung ist überarbeitet worden (kundenspezifisch)

    • [SEP-1891] - Zeiterfassung / Der Mitarbeiter kann seine eigenen Zeitdaten verwalten, mittels separat spezifizierter Rolle (kundenspezifisch)

    • [SEP-1940] - Zeiterfassung / Diverse Regeln (Schicht, Überzeit und Spezialfälle wie Nachtzeitausgleich) für Zeitabrechnung umsetzen (kundenspezifisch)

    • [SEP-1941] - Zeiterfassung / Tagesrapport - 'Fahrzeugplanung' für Tagesrapport (kundenspezifisch)

    • [SEP-1942] - Zeiterfassung / Tagesrapport - Tagesrapport mit Dienstdaten + Fahrzeuginfos als Word-Dokument (kundenspezifisch)

    • [SEP-1943] - Zeiterfassung / Tagesrapport - Bemerkungsfelder im Tagesrapport und Aufrufmöglichkeit Word-Tagesrapport (kundenspezifisch)

    • [SEP-1990] - ALIS / Servicedate als DATE (Aus Einsatzdatum) oder DATETIME (aus Alarmzeitpunkt) ein/auschaltbar

    • [SEP-1759] - Faktura / Einsatzstatus nicht 'abgeschlossen' bei mehreren Rechnungen wird jetzt nach dem Export auf 'abgeschlossen' gesetzt

    • [SEP-1958] - Faktura / Beim 'First Responder' wird der Zuschlag für Nacht- und Feiertag jetzt korrekt verrechnet (kundenspezifisch)

    • [SEP-1962] - SSE-Schnittstelle / Exception bei Übernahme aus Inbox ist behoben (kundenspezifisch)

    • [SEP-1964] - AmbuPad / AmbuPad-Import aus Inbox erzeugt Exception, ist behoben (kundenspezifisch)
