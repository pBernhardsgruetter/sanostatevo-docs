Release Notes - V1.5.5

V1.5.5 - 23. Juni 2017

Fix

  • Einsatzerfassung / Sobald die letzte Rechnung eines Einsatzes mittels Schnittstellen exportiert worden ist, wurde der Erfassungsstatus nicht auf 'Abgeschlossen' gesetzt. SEP-107 / KPSERSG-56

  • Einsatzerfassung / Doppelter Export einer Rechnung, wenn diese am selben Tag storniert und neu erstellt wurde SEP-111 / KPSERSG-55

  • Einsatzverwaltung / Ein abgeschlossener Einsatz kann im Nachhinein nicht abgerechnet werden SEP-127 / KPSEGL-7 / KPSERSG-18

  • Einsatzverwaltung / Ein abgeschlossener Einsatz kann vom Superuser nicht gelöscht werden SEP-119 / KPSEKSA-13 / KPSEGL-1

Enhancements

  • Einsatzerfassung / Der Superuser kann das Einsatzdatum im kontrollierten Einsatz im Nachhinein ändern SEP-122 / KPSERSG-36 Kontrollierter Einsatz - Datum ändern

  • RSG: Einsatzerfassung / Bei S1 Einsätzen darf 'Disponachfrage' kein Pflichtfeld sein SEP-132 / KPSERSG-21 / KPSERSG-47

  • Einsatzverwaltung / Leistungsart 'Notarztbegleitung' hat im GUI das RTW Icon SEP-125 / KPSERSG-8

  • Medizinische Daten / Maxicosi und Kiddy fehlen in den Bergungsarten SEP-114 / KPSEGL-4

  • Medizinische Daten / 'Übergabe REGA' ist keine Lagerung SEP-121 / KPSEGL-5

  • KSGL: Medizinische Daten / GCS und AVPU können nicht beide gleichzeitig erfasst werden SEP-120 / KPSEGL-8

  • KSGL: Medizinische Daten / Diverse Pflichtfelder müssen umgesetzt werden SEP-117 / KPSEGL-10

  • KSGL: Medizinische Daten / Diagnose Tumorleiden fehlt SEP-124 / KPSEGL-3

  • Medizinische Daten / Auswahl Maximalwert der Atemfrequenz auf 150 erhöhen SEP-133 / KPSERSG-33

  • RSG: Faktura / Falscher Tarif bei 'Notarzt für Externe' verwendet SEP-128 / KPSERSG-7

  • RSG: Faktura / Leistungsart 'Legalinspektion' bis anhin nicht spezifiziert, kein Tarifkatalog hinterlegt SEP-129 / KPSERSG-4

  • STS: Faktura / PID wird jetzt auf dem Leistungsbeleg nach FID angezeigt SEP-109 / KPSESTS-23

  • STS: Faktura / Faktura Regeln bezüglich Noteinsatzfahrzeug und Rapid Responder anpassen SEP-134 / KPSESTS-19
