Release Notes - V1.7.3

V1.7.3 - 21. November 2017

Fix

  • Stammdaten / Die Leistungsnummern primär und sekundär werden im GRID angezeigt. SEP-327 / LIMMI

  • Stammdaten / In den Leistungen werden 'Leistungen ohne Preis' mit Hilfe der Validierung verhindert. SEP-338

  • Faktura / UST - Bei Aufgebotsstufe K4 wird der richtige Tarif 'Grundtaxe Transport' angewendet. SEP-340 / KPSEUST-340

  • Faktura / UST - Zuschlag Nacht wird korrekt angewendet SEP-341 / KPSEUST-341

Enhancements

  • Medizinische Daten / Bei der Atmung ist das 'Atemweg-Management' mit dem Eintrag 'Entlastung Pneumothorax' ergänzt worden. SEP-347 / KPSESOH-13

  • Sanoweb / Wird ein Einsatz gespeichert, werden die Patientendaten ans ELS geschickt. SEP-83

  • Sanoweb / Bei der Übernahme des Ereignisortes wird die Strassennummer korrekt übernommen. SEP-324 / KPSEUST-73

  • Sanoweb / Das System von Sanostat Evo ermittelt die PLZ des Ereignisortes aus Sanoweb anhand des Ortes. SEP-343 / KPSEUST-74

  • Einstellungen BST-Admin / Der BST-Administrator kann in den Einstellungen konfigurieren, dass die Retablierungsdauer zu dem Zeitpunkt 'Einsatzende' dazu gerechnet wird. SEP-351 / KPSEUST-76

  • Faktura / STS - Beim 'Rapid Responder' werden Zuschlag Nacht und Zuschlag So/Feiertage abgerechnet. SEP-264 / KPSESTS-52
