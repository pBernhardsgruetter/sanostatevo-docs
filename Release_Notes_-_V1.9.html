    Release Notes - V1.9

    V1.9.0 - 28.02.2018

    Features

    • [SEP-409] - Einsatzerfassung / Adresse aus EO resp. ZO in Patient übernehmen

    • [SEP-589] - Einsatzerfassung / Diagnose als Muss-Feld bei verschiedenen Leistungsarten definiert (kundenspezifische Parametrisierungen für Glarus)

    • [SEP-268] - Med. Daten / Ergänzung Diagnoseliste, Korrektur Tippfehler, Zuordnung "Einteilung"

    • [SEP-375] - Med. Daten / Diagnose mit Aktiv-Von/Bis

    • [SEP-330] - Berichte / Stammdaten - Diagnose, FlexSelect-Filterkriterien erweitern, damit die Diagnoseliste gefiltert werden kann

    • [SEP-473] - Berichte / Ausgabe der Information zu Ereignisort, Abfahrtsort, Zielort verbessert.
      Wenn Typ = Stützpunkt, so wird der Stützpunkt-Kurzname ausgegeben, wenn Typ = Institution/Spital, so wird auch der Kurzname der Institution ausgegeben andernfalls wird PLZ und Ort ausgegeben.
      In einer separaten Spalte wird die Strasse mit Nummer ausgegeben. Diese Anpassung wurde in verschiedenen Berichten angepasst.

    • [SEP-328] - Faktura / Flag "ZusatzTextFuerAdminLeistung" aus Leistung.LeiIstGrundtaxe ermitteln, damit so der Transportweg per ALIS-Schnittstelle übermittelt werden kann

    • [SEP-406] - Faktura / SOH - Zusätzlicher Tarifkatalog TR.94XX-Tarifstuktur umsetzen

    • [SEP-577] - Faktura / SOH - Anzahl Patienten bei Mehrfachtransport als "Anzahlwert" übergeben (kundenspezifische Erweiterung für Soh)

    • [SEP-581] - Faktura / LST - Notarztabrechnung anhand Einsatzzeiten (Tarmed-Regel) für Hinweg- und Besuchszeit

    • [SEP-582] - Faktura / LST - Neues Feld "Notarzt Einsatzende (Verrechnung)" - für Abrechnung LST, Notarzt mit welchem - kundenspezifisch - die Berechnung der Arztdauer für die Fakturierung übersteuert werden kann

    • [SEP-509] - Auswertung / Nicht erreichte Ausrückdauer als Excelliste mit Ausrückzeit, erreicht/nicht erreicht, Besatzung

    • [SEP-538] - Auswertung / Filter für "Summe Einsatzdauer pro Monat und Fahrzeug, Einsatzart"

    • [SEP-540] - Auswertung / Excelliste "Einsatzliste mit Grunddaten (erweitert)" um den Kanton zu Ereignisort erweitert

    • [SEP-551] - Auswertung / Tabelle für die Ausgabe der Anzahl Einsätze pro Mitarbeiter, Aufgebotsstufe und Monat

    • [SEP-552] - Auswertung / Performance für die Statistikauswertung von Ereignissen optimiert

    • [SEP-553] - Auswertung / Filterkriterien für Berichte überarbeitet: Pro Report wurde die zur Verfügung stehenden Filterkriterien für FlexSelect überprüft

    • [SEP-554] - Auswertung / Standardfilter für jeden Bericht definieren: Pro Report wurde definiert, mit welchem Standardfilter gefiltert werden soll (z.B. akt. Monat, nur Primäreinsätze für Hilfsfrist-Report)

    • [SEP-555] - Auswertung / Tabelle für die Ausgabe der Anzahl Einsätze pro Mitarbeiter, Einsatzfunktion und Monat

    • [SEP-556] - Auswertung / Tabelle für die Ausgabe der Rechnungstotal pro Einsatzart und Monat

    • [SEP-557] - Auswertung / Hilfsfristen bezogen auf das Ereignis (beste Hilfsfrist aus Einsätze) mit Ausgabe pro Region (z.B. Spitalregion) mit 90%-Percentile und Erreichnungsgrad

    • [SEP-592] - Auswertung / Excelexport als Liste für Patientenadministration erweitert und diverse Felder und Feldreihenfolgen

    • [SEP-608] - Auswertung / Alle Excelvorlagen überarbeiten und vereinheitlicht (Überschriften, Schriftarten, Formatierungen, Farben)

    • [SEP-620] - Auswertung / Bei allen Auswertungen (Grafiken) wurden Standardfilter gesetzt

    • [SEP-622] - FlexSelect / Neu funktioniert die Filterung bei Stringkriterien mit NOT richtig und sucht auch Leereinträge (NULL-Werte).

    • [SEP-624] - DWH / Syncer (Datenübernahme von Produktiv auf Auswertungsmodul)- Performance optimieren

    • [SEP-626] - DWH / Anzeige des Totalbetrages für Rechnungen wurde überarbeitet (stornierte werden nicht mehr berücksichtigt)

    • [SEP-601] - Sanoweb / Orterkennung bei Ort mit (Kanton) verbessert. Beispiel: Es gibt Ort mit Kantonskürzel, welche unterschiedlich geschrieben werden → Buchs (SG) vs. Buchs vs. Buchs SG. Sanoweb wurde so angepasst, dass Ortsnamen mit und ohne Klammerangaben erkannt werden

    • [SEP-300] - AmbuPad / Einsatzprotokoll PDF an Einsatz (zu Patient) hängen

    • [SEP-386] - AmbuPad / Stammdaten im CSV-Format - Grundkonzept testen

    • [SEP-478] - AmbuPad / Basiseinsatz aus AmbuPad V1.9 International (Testversion Stand Januar 2018 von ZOLL) kann in Sanostat übernommen werden

    • [SEP-503] - AmbuPad / Stammdaten-Export erweitern um 30 zusätzliche Datenbereiche wie Sampler-Auswahlfelder, Begründungen, etc.

    • [SEP-576] - AmbuPad / Konzept für Mehrfachfiles für gleichen Einsatz


    Fix

    • [SEP-472] - Einsatzerfassung / Das Problem, dass PLZ/Ort nicht mehr automatisch bei der Sanoweb-Datenübernahme ins Ereignis befüllt wurden, wurde behoben

    • [SEP-565] - Einsatzerfassung / Die Erfassung eines 2. Einsatzes zu Ereignis löschte den Ereignistyp (Sonst, Institution). Diese wird in der neuen Version verhindert

    • [SEP-587] - Einsatzerfassung / Beim Umschalten bei Ereignisort von "Institution" auf "sonstige" wurde die Adresse nicht gelöscht

    • [SEP-585] - Evo-Nummer / Problematik, dass doppelte SanostatEvo-Nummer vorhanden wurde analysiert und das Erzeugen von Doppeleinträgen mittels Software-Anpassung verhindert

    • [SEP-560] - Rechnungsübersicht / Die Anzeige der Anzahl Rechnungen war abgeschnitten

    • [SEP-641] - Faktura / Workflow lässt Mehrfacherstellung von Rechnungen zu

    • [SEP-566] - Berechtigungen / Rolle "Standarduser" kann abgeschlossene Einsätze wieder öffnen und löschen (Fehler aus V1.8). Mit V1.9 ist dies nicht mehr möglich

    • [SEP-600] - Sanoweb / Diverse Probleme (Spezialfälle von Einsatzdaten) beim Einlesen von Einsätzen wurde ermittelt und korrigiert

    • [SEP-331] - System / Das Löschen von gespeichten Filtern wurde überarbeitet und der Fehler, dass gespeicherte Filter nicht gelöscht werden konnten, wurde eliminiert

    • [SEP-590] - Berichte / Die Scrollbar ist nun wieder am Grid angeheftet

    • [SEP-537] - Auswertung / Anzahl Einsätze pro Monat verteilt nach Einsatzart und mit/ohne Notarzt funktionierte nicht und wurde überarbeitet. Dieser ist nun inhaltlich richtig und performant

    • [SEP-578] - Defaultwerte / Probleme mit Datumsdefaultwerte und SQL-Server (z.B. Stammdaten, Medikamente) als deutsche Version wurde teilweise nicht gesetzt

    • [SEP-612] - FlexSelect / Filter nach Ereignisort-Region lieferte keine Datensätze
